TryHackMe | CTF collection Vol. 1 | Writeup

  • September 16, 2024
  • By Sarang vp

TryHackMe’s CTF collection Vol. 1 is an easy-level room involving decoding, metadata, steganography, PNG magic numbers, OSINT, web archives, and packet analysis to find flags. This writeup will go through each step required to complete the room.

Task 2: What does the base said?

1. Feed me the flag!

Read the above and click the “Completed” button.

Task 1: Author note

1. High five!

Based on the task name and the equal signs after the string, we can assume that this is probably Base64. We can plug this in to CyberChef and confirm our suspicions to get the flag.

Task 3: Meta meta

1. I’m hungry, I need the flag.

Opening the image shows a picture of the earth. Nothing here stands out as the flag, so let’s dig into the metadata of the image. In our terminal, we can use ExifTool to see the metadata of the image. We’ll use the following command: $ exiftool Findme.jpg

Task 4: Mon, are we going to be okay?

1. It is sad. Feed me the flag.

This image has nothing interesting in it’s metadata. Let’s use Steghide to look for anything hidden using the following command: $ steghide --info Extinction.jpg

When asked for a password we simply hit enter which reveals that there is a hidden text file.

We can now use the following command to extract the file: $ steghide extract -sf Extinction.jpg

When asked for a passphrase we hit enter and see that the data is written to a file.

Use cat on the file reveals the flag to us.

Task 6: QRrrrr

1. More flag please!

The image we download is a QR code. Scanning the code leads to a Google search of the flag.

Task 7: Reverse it or read it?

1. Found the flag? After downloading the file, we can open it in Notepad. We’re met with a lot of random symbols, scrolling down we can find the flag.

Task 8: Another decoding stuff

1. Oh, Oh, Did you get it? This is another encoded string that we need to run through CyberChef. We can use the Magic recipe to see if it can automatically decode it. We can also just play around with different encoding structures until we find the correct one. Using Base58, we find that the flag becomes readable.

Task 9: Left or right

1. What did you get? The task says that ROT13 is too mainstream. ROT13 is a Caesar Cipher, let’s plug the string into dcode.fr’s Caesar Cipher decoder to see if it can solve it. We find that it is a Caesar Cipher and the solve for it.

Task 10: Make a comment

1. I’m hungry now… I need the flag This is similar to task 5. There is nothing visible in the task. If we inspect the page we can find the flag in the code of the task.

Task 11: Can you fix it?

1. What is the content? If we download and try to open the image, we get an error message. In some CTFs it is common to be given an image with code changed so the file cannot be opened. This code is called “magic numbers” and it is how computers read file types. We can go to hexed.it to see the magic numbers of the file we were given. These values do not match the ones required for a PNG image. Let’s change those magic numbers to the correct values. change to the correct values We are now able to open the image and see the flag.

Task 12: Read it

1. Did you found the hidden flag? This one is pretty difficult. The flag is in a post on the TryHackMe subreddit. It can be found by searching “tryhackme room reddit” and is in a post titled “New room Coming soon!”.

Task 13: Spin my head

1. Can you decode it? This is a programming language called Brainfuck. We can put this code into dcode.fr’s Brainfuck interpreter to get the flag.

Task 14: An exclusive!

1. Did you crack it? Feed me now! We’re given two strings, based on the task name we can assume they will be used in an exclusive or (XOR) operation. We can find one online and play with the settings until the flag is revealed.

Task 15: Binary walk!

1. Flag! Flag! Flag! The task name is called binary walk, which implies we need to use Binwalk on the downloaded image. We can use the following command to look for anything with Binwalk: $ binwalk hell.jpg This reveals a text file in the image. Since we know there is a file, we can use the following command to extract it from the image: $ binwalk -e hell.jpg This will create a directory, if we open it and look inside we’ll see the text file. We can then open it to get the flag.

Task 16: Darkness!

1. What does the flag said? The image we’re given look to be a black rectangle. If we run the image through Aperi’Solve, we can see that the flag has been superimposed in the image.

Task 17: A sounding QR

1. What does the bot said? The image attached is another QR code. This one leads to a SoundCloud song with a TTS voice reading the flag. I would recommend using the loop option because the voice is slightly hard to get used to and understand.

Task 18: Dig up the past

1. Did you found my past? We’re given the URL of a website and a date we need to view it at. We can do this using the "Wayback Machine". If we go to the specified date we’ll see that a snapshot has been taken.

Checking the snapshot shows us an uncategorized article with the flag in it.

Task 19: Uncrackable!

1. The deciphered text This is a Vigenere Cipher, which is similar to the Caesar Cipher but uses multiple values to shift instead of one. For a good explanation, click here. We’re given the format for this flag, which actually makes it a pretty easy solve. We can use "dcode.fr’s Vigenere Cipher tool" to solve it with a known plaintext word.

Task 20: Small bases!

1. What is the flag? I was using CyberChef to solve this and kept getting the wrong answer. We have to convert from a Decimal number to Hexadecimal and then to ASCII. I ended up having to use RapidTables to do the conversions.

Task 21: Read the packet

1. Did you captured my neighbor’s flag? Since we know the format of the flags, we can simply do a string search for “THM{“. This finds a packet that has the flag in it.

We’ve completed the room!
